# daily/ — 值班日记(本地专用)

> ⚠️ 这个目录下的所有 `*.md` 文件**永远不上传 GitHub**(由 `.gitignore` 强制)。
> 这里是写真实病例细节的地方——患者隐私靠目录隔离,不靠记性。

## 工作流

每个值班日(或每天)新建一个文件:`2026-04-27.md`,使用 `_templates/值班日记模板.md`。

## 文件如何与公开内容互动

- 在 `daily/2026-04-27.md` 里可以引用疾病页:`今日学到 [[急性冠脉综合征]] 不典型表现`
- 反向链接会出现在 `急性冠脉综合征.md` 底部 → 但**只在本地可见**(daily 不上传)
- 公开访问者看不到你的具体病例,但你自己拥有完整知识图谱

## 即使在本地也要的纪律

即使不上传,也建议:
- 不写患者全名,用"32M with DM2"格式
- 不写确切日期 + 床号组合
- 备份这个文件夹要选加密手段(Time Machine 加密、本地加密硬盘)
- 离职 / 毕业时考虑销毁该目录

## 想公开某个特别有教育价值的"匿名病例"?

流程:
1. 在 `daily/` 里写完原始记录
2. 复制一份到 `examples/` 或对应疾病文件
3. **彻底重写**,去除所有可识别信息
4. 引用方式:"曾遇到一例类似情况:..." 而非"2026-04-27 床 12 患者"
5. 让一位同学读一遍,问"你能猜到是谁吗"——如果能,继续模糊化
